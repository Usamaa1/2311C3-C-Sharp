## üü¢ Generic vs Non-Generic

### üî¥ Non-Generic Example

### ‚úÖ Generic Example
```csharp

// numbers.Add("Hello"); // Compile-time error

```

### ‚úÖ Generic Method
```csharp

{
    Console.WriteLine($"Value: {value}");
}

```

# üë∂ Object-Oriented Programming (OOP) Concepts in C# (C# 9+)

## üîπ 1. Class and Object

**Real-life Example**: A *Class* is like a *blueprint* of a house. An *Object* is the actual house built from that blueprint.

```csharp
// Class (Blueprint)
public class Car
{
    public string Brand;
    public int Speed;

    public void Drive()
    {
        Console.WriteLine($"Driving {Brand} at {Speed} km/h");
    }
}

// Object (Real car)
var myCar = new Car { Brand = "Toyota", Speed = 80 };
myCar.Drive();
```

---

## üîπ 2. Encapsulation (Hiding Data)

**Real-life Example**: A TV remote hides the complex circuit inside. You only use buttons.

```csharp
public class BankAccount
{
    private double balance = 0;

    public void Deposit(double amount)
    {
        if (amount > 0) balance += amount;
    }

    public double GetBalance()
    {
        return balance;
    }
}
```

---

## üîπ 3. Inheritance (Is-a Relationship)

**Real-life Example**: A *Dog* is an *Animal*.

```csharp
public class Animal
{
    public void Eat() => Console.WriteLine("Eating");
}

public class Dog : Animal
{
    public void Bark() => Console.WriteLine("Barking");
}

Dog dog = new Dog();
dog.Eat(); // from Animal
dog.Bark();
```

---

## üîπ 4. Polymorphism (Many Forms)

**Real-life Example**: A remote control can operate TV, AC, or Music Player ‚Äî same remote, different behavior.

```csharp
public class Animal
{
    public virtual void Speak() => Console.WriteLine("Animal sound");
}

public class Cat : Animal
{
    public override void Speak() => Console.WriteLine("Meow");
}

public class Dog : Animal
{
    public override void Speak() => Console.WriteLine("Woof");
}

Animal myPet = new Cat();
myPet.Speak(); // Meow
```

---

## üîπ 5. Abstraction (Only Essential Info)

**Real-life Example**: You drive a car without knowing how the engine works internally.

```csharp
public abstract class Vehicle
{
    public abstract void Start(); // only method signature
}

public class Bike : Vehicle
{
    public override void Start() => Console.WriteLine("Bike started");
}

Vehicle vehicle = new Bike();
vehicle.Start();
```

## üîπ 6. Interface (Contract)

**Real-life Example**: A *USB* interface ‚Äî any device (mouse, keyboard) can implement it.

```csharp
public interface IAnimal
{
    void MakeSound();
}

public class Cow : IAnimal
{
    public void MakeSound() => Console.WriteLine("Moo");
}

IAnimal animal = new Cow();
animal.MakeSound();
```

üëâ Interfaces define *what* a class must do, not *how*.

## üîπ 11. Generics vs Non-Generics

**Real-life Example**: A *lunchbox* (Generic) that works for any type of food.

# C# 9: Getters, Setters, and `init` Properties (Immutability)

This guide explains the use of **getters**, **setters**, and the **`init` accessor** introduced in **C# 9**, with real-world examples and comparisons.

---

## ‚úÖ Traditional Getter and Setter

```csharp
public class Student
{
    private string name;

    public string Name
    {
        get { return name; }
        set { name = value; }
    }
}
```

**Usage:**

```csharp
var student = new Student();
student.Name = "Alice";
Console.WriteLine(student.Name); // Output: Alice
```

---

## ‚úÖ Auto-Implemented Properties (C# 9 Compatible)

```csharp
public class Product
{
    public int Id { get; set; }
    public string Name { get; set; }
}
```

**Usage:**

```csharp
var product = new Product { Id = 101, Name = "Laptop" };
Console.WriteLine($"ID: {product.Id}, Name: {product.Name}");
```

---

## ‚úÖ Read-Only Property (Only Getter)

```csharp
public class Circle
{
    public double Radius { get; set; }

    public double Area
    {
        get { return Math.PI * Radius * Radius; }
    }
}
```

**Usage:**

```csharp
var circle = new Circle { Radius = 5 };

```

---

## ‚úÖ Getters and Setters with Logic

```csharp
public class Employee
{
    private int salary;

    public int Salary
    {
        get { return salary; }
        set
        {
            if (value >= 0)
                salary = value;

        }
    }

```

**Usage:**

```csharp

emp.Salary = 50000;
```

## üÜï C# 9 Feature: `init` Accessor

### ‚úÖ What is Immutable?

**Immutable** means:

> ‚ùå *Cannot be changed after creation.*

Once a property is set, it **cannot be modified** later.

---

### ‚úÖ What is `init`?

`init` is a new accessor introduced in **C# 9**.
It allows setting a property **only once during object creation**.

```csharp
public class Person
{
    public string Name { get; init; }
}
```

**Usage:**

```csharp
var p = new Person { Name = "Alice" }; // ‚úÖ Allowed
// p.Name = "Bob"; // ‚ùå Compile-time error
```

---

## ‚úÖ Example with Constructor and `init`

```csharp
public class Car
{
    public string Model { get; init; }
    public int Year { get; init; }

    public Car(string model)
    {
        Model = model;
    }
}
```

**Usage:**

```csharp
var car = new Car("Toyota") { Year = 2021 };
Console.WriteLine($"Model: {car.Model}, Year: {car.Year}");
```

---

## üü® Difference Between `set` and `init`

| Feature               | `set`        | `init`            |
| --------------------- | ------------ | ----------------- |
| Can modify later      | ‚úÖ Yes        | ‚ùå No              |
| Set during init       | ‚úÖ Yes        | ‚úÖ Yes             |
| Supports immutability | ‚ùå No         | ‚úÖ Yes             |
| Use case              | Mutable data | Immutable objects |

---

## ‚úÖ Bonus: `record` Type + `init`

```csharp
public record Book
{
    public string Title { get; init; }
    public string Author { get; init; }
}
```

**Usage:**

```csharp
var book = new Book { Title = "1984", Author = "George Orwell" };
// book.Title = "Animal Farm"; // ‚ùå Not allowed
```

---

## üìå Summary

* Use `get`/`set` for **mutable** properties.
* Use `init` for **immutable** properties set only during initialization.
* `init` helps create **safe, predictable, thread-safe** code.
* Combine `init` with `record` types for **concise, immutable models** in C# 9.
